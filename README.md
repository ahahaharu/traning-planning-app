## Функциональные требования

#### Общее описание участников и случаев использования:
 - **Акторы:** Пользователь (человек, взаимодействующий с веб-приложением для планировки тренировок).
 - **Система:** Программное средство для планировки фитнес тренировок
 - **Описание:** Веб-приложение для регистрации пользователей, создания и управления упражнениями, программами тренировок, а также отслеживания прогресса тренировок и параметров тела.

#### Случаи использования:

1. Зарегистрироваться в приложении
2. Войти в систему
3. Редактировать профиль
4. Создать упражнение
5. Редактировать упражнение
6. Удалить упражнение
7. Создать программу тренировок
8. Редактировать программу тренировок
9. Удалить программу тренировок
10. Записать выполненную тренировку
11. Просмотреть историю тренировок
12. Просмотреть статистику прогресса
13. Просмотреть список упражнений
14. Просмотреть список программ тренировок
15. Добавить упражнение в программу
16. Удалить упражнение из программы

#### Описание Use Cases

##### Use Case 1: Зарегистрироваться в приложении
  - **Цель:** Создать новую учётную запись пользователя.
  - **Акторы:** Пользователь
  - **Предусловие:** Пользователь не зарегистрирован, приложение запущено.
  - **Основной сценарий:**
      1. Пользователь переходит на страницу регистрации.
      2. Система отображает форму регистрации.
      3. Пользователь вводит имя, email, пароль, рост и вес.
      4. Пользователь нажимает кнопку "Зарегистрироваться".
      5. Система проверяет уникальность email.
      6. Система сохраняет данные пользователя в базу данных.
      7. Система перенаправляет пользователя на страницу входа.
  - **Исключения:**
      - Если email уже используется, система выводит сообщение "Email занят" и просит ввести другой.
      - Если поля заполнены некорректно (например, рост/вес не числа), система выводит сообщение об ошибке.

##### Use Case 2: Войти в систему
  - **Цель:** Получить доступ к учётной записи.
  - **Акторы:** Пользователь
  - **Предусловие:** Пользователь зарегистрирован, приложение запущено.
  - **Основной сценарий:**
      1. Пользователь переходит на страницу входа.
      2. Система отображает форму входа.
      3. Пользователь вводит email и пароль.
      4. Пользователь нажимает кнопку "Войти".
      5. Система проверяет email и пароль в MongoDB.
      6. Система перенаправляет пользователя на главную страницу.
  - **Исключения:**
      - Если email или пароль неверны, система выводит сообщение "Неверные данные" и просит повторить ввод.
   
##### Use Case 3: Редактировать профиль
  - **Цель:** Обновить данные профиля пользователя.
  - **Акторы:** Пользователь
  - **Предусловие:** Пользователь вошёл в систему.
  - **Основной сценарий:**
      1. Пользователь переходит на страницу профиля.
      2. Система отображает текущие данные (имя, email, рост, вес).
      3. Пользователь изменяет нужные поля.
      4. Пользователь нажимает кнопку "Сохранить".
      5. Система обновляет данные в базе данных.
      6. Система отображает сообщение "Профиль обновлён".
  - **Исключения:**
      - Если данные некорректны (например, вес отрицательный), система выводит сообщение об ошибке.

##### Use Case 4: Создать упражнение
  - **Цель:** Добавить новое упражнение в базу.
  - **Акторы:** Пользователь
  - **Предусловие:** Пользователь вошёл в систему.
  - **Основной сценарий:**
      1. Пользователь переходит на страницу управления упражнениями.
      2. Пользователь нажимает кнопку "Добавить упражнение".
      3. Система отображает форму создания упражнения.
      4. Пользователь вводит название, тип (силовое/кардио/гибкость) и описание.
      5. Пользователь нажимает "Сохранить".
      6. Система сохраняет упражнение в базе данных с привязкой к пользователю (`ownerId`).
      7. Система обновляет список упражнений пользователя.
  - **Исключения:**
      - Если обязательные поля пусты, система выводит сообщение об ошибке.
      - Если данные некорректны, система выводит сообщение об ошибке.

##### Use Case 5: Редактировать упражнение

##### Use Case 6: Удалить упражнение
  - **Цель:** Удалить упражнение из базы.
  - **Акторы:** Пользователь
  - **Предусловие:** Пользователь вошёл в систему, упражнение существует.
  - **Основной сценарий:**
      1. Пользователь переходит на страницу управления упражнениями.
      2. Пользователь выбирает упражнение и нажимает "Удалить".
      3. Система запрашивает подтверждение удаления.
      4. Пользователь подтверждает действие.
      5. Система удаляет упражнение из базы данных и обновляет список.
  - **Исключения:**
      - Если упражнение используется в программах тренировок, система предупреждает об этом.

#### Примечания
  - Все действия начинаются с главной страницы или навигации по вкладкам.
  - Данные сохраняются в базе данных.
  - Приложение работает в режиме реального времени, пока пользователь не выйдет из системы.
